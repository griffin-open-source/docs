## Overview

The Griffin Runner provides a REST API for managing and monitoring API tests in production. Use this API to deploy tests, view execution results, trigger manual executions, and manage test configurations.

## Base URL

```
https://runner.example.com/api
```

<Warning>
  Replace `runner.example.com` with your actual runner service host. The base URL can be configured using the CLI: `griffin configure-runner-host <host>`
</Warning>

## Authentication

All API endpoints require authentication using API keys or bearer tokens.

### Using API Keys

Include your API key in the `X-API-Key` header:

```http
X-API-Key: your-api-key-here
```

### Using Bearer Tokens

Include your bearer token in the `Authorization` header:

```http
Authorization: Bearer your-token-here
```

## Response Format

All API responses are in JSON format. Successful responses use standard HTTP status codes:

- `200 OK` - Request succeeded
- `201 Created` - Resource created successfully
- `204 No Content` - Request succeeded, no content to return
- `400 Bad Request` - Invalid request
- `401 Unauthorized` - Authentication required
- `404 Not Found` - Resource not found
- `500 Internal Server Error` - Server error

### Error Response Format

Error responses follow this structure:

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "details": {}
  }
}
```

## Rate Limiting

API requests are rate-limited to prevent abuse. Rate limit headers are included in responses:

```http
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
```

## Examples

### Using cURL

```bash
# List all tests
curl -X GET https://runner.example.com/api/tests \
  -H "X-API-Key: your-api-key"

# Deploy a test
curl -X POST https://runner.example.com/api/tests \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d @test-plan.json
```

### Using the CLI

The Griffin CLI provides a convenient interface to the API:

```bash
# Configure API endpoint
griffin configure-runner-host https://runner.example.com

# Deploy tests
griffin deploy

# View logs
griffin logs my-check
```
