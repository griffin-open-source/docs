## Get started in three steps

Create your first API test and run it locally against your development server.

### Step 1: Install Griffin CLI

<AccordionGroup>
  <Accordion icon="download" title="Install via npm">
    Install the Griffin CLI globally using npm:
    
    ```bash
    npm install -g griffin-cli
    ```
    
    Or use `npx` to run it without installing:
    
    ```bash
    npx griffin-cli run-local
    ```
  </Accordion>
  <Accordion icon="code-branch" title="Build from source">
    If you're working with the Griffin repository directly:
    
    ```bash
    cd griffin-cli
    npm install
    npm run build
    ```
    
    Then run the CLI from the build directory:
    
    ```bash
    node dist/cli.js run-local
    ```
  </Accordion>
</AccordionGroup>

### Step 2: Create your first test

<AccordionGroup>
  <Accordion icon="file-code" title="Create a test file">
    Create a test file in a `__griffin__` subdirectory:
    
    ```bash
    mkdir -p __griffin__
    touch __griffin__/my-first-test.ts
    ```
    
    <Tip>Griffin automatically discovers `.ts` test files in `__griffin__` subdirectories.</Tip>
  </Accordion>
  <Accordion icon="pen-to-square" title="Write your test">
    Add the following code to your test file:

    ```typescript
    import { createTestBuilder, GET, JSON, Assert, Frequency } from "griffin-ts";

    const monitor = createTestBuilder({
      name: "my-first-check",
      endpoint_host: "http://localhost:3000"  // Your API server URL
    })
      .endpoint("health", {
        method: GET,
        response_format: JSON,
        path: "/health"
      })
      .assert((state) => [
        Assert(state["health"].status).equals(200)
      ])
      .build({ frequency: Frequency.every(1).minute() });

    export default monitor;
    ```

    <Warning>Make sure to update `endpoint_host` with your actual API server URL and port.</Warning>
  </Accordion>
</AccordionGroup>

### Step 3: Run your test

<AccordionGroup>
  <Accordion icon="play" title="Start your API server">
    Make sure your API server is running. For example, if using the sample API:
    
    ```bash
    cd sample-apis
    npm install
    npm run dev
    ```
    
    This starts the server on `http://localhost:3000` (or your configured port).
  </Accordion>
  <Accordion icon="rocket" title="Run tests locally">
    First, initialize Griffin in your project and create an environment:

    ```bash
    griffin init
    griffin env add dev
    ```

    Then run all discovered tests for the dev environment:

    ```bash
    griffin local run dev
    ```

    Or if using npx:

    ```bash
    npx griffin-cli local run dev
    ```

    Griffin will discover all test files in `__griffin__` directories and execute them against their configured endpoint hosts.
  </Accordion>
</AccordionGroup>
