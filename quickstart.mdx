## Get started in three steps

Create your first API monitor and run it locally against your development server.

### Step 1: Install Griffin CLI

<AccordionGroup>
  <Accordion icon="download" title="Install via npm">
    Install the Griffin CLI globally using npm:
    
    ```bash
    npm install -g @griffin-app/griffin
    ```
    
    Or use `npx` to run it without installing:
    
    ```bash
    npx @griffin-app/griffin run-local
    ```
  </Accordion>
  <Accordion icon="code-branch" title="Build from source">
    If you're working with the Griffin repository directly:
    
    ```bash
    cd @griffin-app/griffin
    npm install
    npm run build
    ```
    
    Then run the CLI from the build directory:
    
    ```bash
    node dist/cli.js run-local
    ```
  </Accordion>
</AccordionGroup>

### Step 2: Create your first monitor

<AccordionGroup>
  <Accordion icon="file-code" title="Create a monitor file">
    Create a monitor file in a `__griffin__` subdirectory:
    
    ```bash
    mkdir -p __griffin__
    touch __griffin__/my-first-monitor.ts
    ```
    
    <Tip>Griffin automatically discovers `.ts` monitor files in `__griffin__` subdirectories.</Tip>
  </Accordion>
  <Accordion icon="pen-to-square" title="Write your monitor">
    Add the following code to your monitor file:

    ```typescript
    import { createMonitorBuilder, GET, JSON, Assert, Frequency } from "@griffin-app/griffin";

    const monitor = createMonitorBuilder({
      name: "my-first-check",
      endpoint_host: "http://localhost:3000"  // Your API server URL
    })
      .endpoint("health", {
        method: GET,
        response_format: JSON,
        path: "/health"
      })
      .assert((state) => [
        Assert(state["health"].status).equals(200)
      ])
      .build({ frequency: Frequency.every(1).minute() });

    export default monitor;
    ```

    <Warning>Make sure to update `endpoint_host` with your actual API server URL and port.</Warning>
  </Accordion>
</AccordionGroup>

### Step 3: Run your monitor

<AccordionGroup>
  <Accordion icon="play" title="Start your API server">
    Make sure your API server is running. For example, if using the sample API:
    
    ```bash
    cd sample-apis
    npm install
    npm run dev
    ```
    
    This starts the server on `http://localhost:3000` (or your configured port).
  </Accordion>
  <Accordion icon="rocket" title="Run monitors locally">
    First, initialize Griffin in your project and create an environment:

    ```bash
    griffin init
    griffin env add dev
    ```

    Then run all discovered monitors for the dev environment:

    ```bash
    griffin local run dev
    ```

    Or if using npx:

    ```bash
    npx @griffin-app/griffin local run dev
    ```

    Griffin will discover all monitor files in `__griffin__` directories and execute them against their configured endpoint hosts.
  </Accordion>
</AccordionGroup>
